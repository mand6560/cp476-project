CP476 Internet Computing Project

Objectives

  • Identify and propose an Internet computing application project
  • Design and implement a multi-tier web application
  • Apply course covered tools to the project development
  • Project packaging, delivery, presentation, and demonstration

The project can be done individually. It can also be done by a group of at most three members. In case of group project, it needs to specify names of all team members, and roles and tasks of each member in the milestone and schedule session of the proposal.

Project topics

You are required to propose your own project topic. The topic can be either a multi-tier Internet/web application, or a tool kit/framework for certain types of Web applications. For Web applications, it can be an information management & service system such as online store, online game, content management system. For tool kit/framework, it can be Internet-based working environment, writing system, framework for personal Web information system. You should do a research before determining your topic. The following are examples of acceptable topics.

  1. A specific web application. For example, Internet based personal library system, working log system, personal data collect and service system, Internet based computer aided reading system. This is a personal system that can save, retrieve and search on-line articles, news, and read and make node, book marks.
  2. Internet based editor, allow multiple people edit a document simultaneously. A drawing program allowing multiple to work on the same drawing together.
  3. Mobile device based Internet computer applications for Mobile devices. Possibly a joint project with mobile computing courses, with focus on sever side processing.
  4. Web system for special needs, such as for teachers to manage teaching materials, and knowledge base.
  5. Web application with graphics interface, multi-player 3D games, using WebGL, WebSocket.
  6. Web based information management and service applications such as online stores, bookstores, art gallery, etc.
  7. Design and implement personalized web framework, a system that can be configured to create customized personal web system. It is allowed to use existing frameworks for specific types of web systems.

Project stages

  1. Project proposal. Refer this project proposal template for components and structure of the proposal.
  2. System design and implementation. This is to develop a prototype of your project to demonstrate your project. Refer to readme file for a breakdown of the this stage of evaluation items.
  3. Project packaging, deployment, demonstration, documentation and presentation.
  4. Project report, evaluation, and submission

Project submission

Check readme.txt for mark distributions and grading items of stages.

It is required to write and submit the project report project_report.md in markdown format as part of project documentation. Take screen shots on the major features and new features of your project, and link the images in the project report.

Project submission include:

  1. Download readme.txt into the project folder. Sign the readme.txt file by filling in your Name, ID, Email, and the self-evaluation for each grading item.
  2. Zip the project folder to create a zip file, and renamed it to CP476_project_YourWLUEmailID.zip. The zip file should contain the following parts.
  • a valid readme.txt file with signatures and self-evaluation
  • revised proposal.pdf, this revised proposal will be used in grading.
  • project report files project_report.md, project_report.html, images folder. Basically, this report is a summary of the project implementation against the proposal. Specifically, it provides a short description / proofs for the self-evaluation on part 2 (as follows) of evaluation grid.
P2 System design & implementation
P2.1 Client side component & UI        
P2.2 Server side CGI components    
P2.3 Database tier design, data, usage  
P2.4 New features and tools  
P2.5 Problem solving algorithms  
P2.6 Efficiency and robustness
  • program folders, including source code programs of your project, Do not include node_modules if you use NodeJs.
  • A project README.md file to describe how to compile, deploy and run your project, as well as external resources used.
  • One presentation document file, which can be a word, ppt slide, or a short video (maximum 10 minutes) file. The project presentation is short introduction and high light of your project to general audiences. If you present your project in class, you can omit the presentation file.

Note that the above zip file serves as a project delivery, it should can contains necessary data, files and documentations, so that marker is able to install, test and verify your project specifications and claims.

  1. Submit the CP476_project_WLUEmailID.zip to project dropbox on MyLS by one team member. For group project, other team members just submit the readme.txt file. One won’t get grade if the readme.txt file is not submitted.
